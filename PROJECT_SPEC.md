# Abyss Survivor PROJECT_SPEC.md

## 1. 프로젝트 개요

`Abyss Survivor`는 모바일 웹 기반의 세로형 미니 생존 로그라이트 게임이다.

이 프로젝트는 이전 프로젝트인 `Abyss Breaker`와 별개의 새 프로젝트로 진행한다.
목표는 복잡한 RPG, 인벤토리, 장비 파밍 게임이 아니라 짧게 반복 플레이할 수 있는 자동공격 생존 게임을 만드는 것이다.

핵심 방향은 다음과 같다.

```text
모바일 세로형
자동공격
간단한 이동 조작
경험치 획득
레벨업 능력 선택
짧은 런
보스 처치
반복 플레이
```

v0.1에서는 기능 확장보다 안정성과 실제 플레이 가능성을 우선한다.

## 5. 전역 네임스페이스

모든 파일은 하나의 전역 네임스페이스 아래에 기능을 등록한다.

```javascript
window.AbyssSurvivor = window.AbyssSurvivor || {};
```

각 파일은 다음 모듈을 담당한다.

```text
data.js   → AbyssSurvivor.Data
state.js  → AbyssSurvivor.State
game.js   → AbyssSurvivor.Game
render.js → AbyssSurvivor.Render
ui.js     → AbyssSurvivor.UI
main.js   → AbyssSurvivor.Main
```

전역 변수 오염을 줄이기 위해 각 파일은 즉시 실행 함수 형태로 작성한다.

```javascript
(function () {
  window.AbyssSurvivor = window.AbyssSurvivor || {};
  const AS = window.AbyssSurvivor;

  AS.Data = {
    // ...
  };
})();
```

## 8. 기본 게임 흐름

v0.1의 기본 흐름은 다음과 같다.

```text
1. 플레이어가 캐릭터를 이동한다.
2. 캐릭터는 가장 가까운 적을 자동 공격한다.
3. 적을 처치하면 경험치 보석이 떨어진다.
4. 보석을 먹으면 경험치가 오른다.
5. 경험치가 가득 차면 레벨업한다.
6. 레벨업 시 게임이 일시 정지되고 능력 3개가 표시된다.
7. 능력 1개를 선택하면 게임이 재개된다.
8. 시간이 지날수록 적 생성 빈도가 증가한다.
9. 일정 시간 이후 보스가 등장한다.
10. 플레이어 체력이 0이 되면 게임오버.
11. 보스를 처치하면 런 클리어.
12. 결과에 따라 간단한 저장 데이터를 갱신한다.
```

## 9. 플레이어 규칙

플레이어는 원형 캐릭터로 표현한다.

초기값은 다음을 기준으로 한다.

```text
체력: 100
최대 체력: 100
이동 속도: 기본값
공격력: 기본값
공격 속도: 기본값
투사체 속도: 기본값
투사체 크기: 기본값
경험치 획득 범위: 기본값
피격 후 짧은 무적 시간 보유
```

피격 무적 시간은 연속 충돌 피해로 즉시 사망하지 않게 하기 위한 안전장치다.

## 10. 조작 규칙

모바일과 PC를 모두 지원한다.

### 모바일

```text
화면을 터치한 뒤 드래그하면 해당 방향으로 이동한다.
손을 떼면 이동을 멈춘다.
```

### PC

v0.1에서는 다음 중 하나 이상을 지원한다.

```text
마우스 드래그 이동
WASD 이동
```

권장 구현은 다음과 같다.

```text
모바일/PC 공통: pointer event 기반 드래그 이동
PC 추가: WASD 이동
```

조작 좌표는 Canvas 표시 크기와 내부 좌표가 어긋나지 않도록 변환한다.

---

## 11. 공격 규칙

플레이어는 자동 공격을 한다.

```text
가장 가까운 적을 자동 조준한다.
공격 쿨타임이 끝나면 투사체를 발사한다.
투사체는 목표 방향으로 직선 이동한다.
투사체가 적에게 닿으면 피해를 준다.
화면 밖으로 나간 투사체는 제거한다.
```

v0.1에서는 투사체 관통, 광역 피해, 유도탄, 속성 공격은 구현하지 않는다.

---

## 12. 적 규칙

v0.1의 적은 다음 4종이다.

```text
일반 적
빠른 적
튼튼한 적
보스
```

적은 화면 가장자리 바깥에서 생성된다.

```text
생성 위치: Canvas 바깥쪽 가장자리
이동 방식: 플레이어를 향해 직선 추적
피해 방식: 플레이어와 충돌 시 피해
제거 조건: 체력 0 이하 또는 비정상 위치
```

보스는 v0.1에서 복잡한 패턴 없이 큰 체력과 큰 크기만 가진다.

---

## 13. 경험치 규칙

적을 처치하면 경험치 보석을 드롭한다.

```text
일반 적: 낮은 경험치
빠른 적: 낮거나 중간 경험치
튼튼한 적: 중간 경험치
보스: 런 클리어 대상
```

보석 획득 규칙은 다음과 같다.

```text
플레이어가 보석 근처에 접근하면 획득한다.
획득 시 경험치가 증가한다.
경험치가 필요량 이상이면 레벨업한다.
레벨업 후 남는 경험치는 다음 레벨에 반영한다.
```

v0.1에서는 보석이 자동으로 빨려 들어오는 복잡한 연출은 필수 구현 대상이 아니다.
다만 획득 범위 증가 능력을 위해 보석 획득 반경 값은 상태값으로 관리한다.

---

## 14. 레벨업 능력 규칙

레벨업 시 게임을 일시 정지하고 능력 3개를 표시한다.

사용자는 3개 중 1개를 선택한다.

v0.1 능력 목록은 다음과 같다.

```text
공격력 증가
공격 속도 증가
투사체 속도 증가
투사체 크기 증가
최대 체력 증가
체력 회복
이동 속도 증가
경험치 획득 범위 증가
```

능력은 수치 강화 중심으로 구현한다.
복잡한 조건부 효과나 조합 효과는 v0.1에서 제외한다.

능력 선택 후 게임은 즉시 재개된다.


## 17. Canvas 크기와 좌표 기준

Canvas의 내부 좌표와 화면 표시 크기가 어긋나면 안 된다.

권장 기준은 다음과 같다.

```text
게임 내부 기준 너비: 360
게임 내부 기준 높이: 560
실제 표시 크기: CSS로 모바일 화면에 맞춤
입력 좌표: 표시 크기 대비 내부 좌표로 변환
```

Canvas 리사이즈 시 다음을 지킨다.

```text
내부 좌표계는 안정적으로 유지한다.
마우스/터치 입력 좌표는 getBoundingClientRect() 기준으로 변환한다.
플레이어 위치는 Canvas 범위 안에 유지한다.
```

## 18. 게임 상태

게임 상태는 명확히 분리한다.

```text
title      : 시작 전
running    : 진행 중
paused     : 일시정지
levelup    : 능력 선택 중
gameover   : 게임오버
clear      : 런 클리어
```

게임 루프는 상태에 따라 업데이트를 제어한다.

```text
running 상태에서만 전투 로직을 업데이트한다.
paused, levelup, gameover, clear 상태에서는 전투 로직을 업데이트하지 않는다.
렌더링은 계속 가능하다.
```

## 19. localStorage 저장

v0.1에서는 간단한 저장만 구현한다.

저장 항목은 다음 정도로 제한한다.

```text
최고 생존 시간
최고 처치 수
누적 플레이 횟수
누적 클리어 횟수
```

저장 키는 다음을 사용한다.

```text
abyssSurvivorSave
```

localStorage 오류가 발생해도 게임 실행을 막으면 안 된다.

```text
저장 실패 시 콘솔 경고만 남긴다.
불러오기 실패 시 기본값으로 시작한다.
저장 데이터가 깨져 있어도 기본값으로 복구한다.
```

## 20. 안정성 기준

v0.1 개발 시 다음 기준을 반드시 지킨다.

```text
게임이 실제로 플레이 가능해야 한다.
모바일 세로 화면에서 조작 가능해야 한다.
Canvas 크기와 실제 좌표가 어긋나면 안 된다.
게임 루프가 중복 실행되면 안 된다.
이벤트 리스너가 중복 등록되면 안 된다.
NaN, Infinity가 발생하면 안 된다.
localStorage 저장 오류가 게임을 막으면 안 된다.
적, 투사체, 보석은 화면 밖에서 적절히 제거해야 한다.
초기 버전에서는 기능보다 안정성을 우선한다.
```
