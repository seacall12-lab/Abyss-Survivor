# Abyss Survivor

`Abyss Survivor`는 모바일 세로 화면 기준의 싱글 플레이 생존 로그라이트 웹게임입니다. 순수 HTML, CSS, JavaScript만 사용하며 외부 라이브러리, ES Module, npm, 번들러 없이 GitHub Pages 루트에서 바로 실행됩니다.

## 핵심 흐름

- 클래스, 구역, 도전 모드, 영구 성장을 선택합니다.
- Canvas를 드래그하거나 WASD/방향키로 이동합니다.
- 플레이어는 가장 가까운 적을 자동 공격합니다.
- 경험치 보석을 획득해 레벨업 능력, 진화, 유물을 선택합니다.
- 보스를 처치하거나 제한 시간까지 생존해 런을 마칩니다.
- 결과에 따라 심연 조각을 얻고 다음 런의 영구 성장에 사용합니다.

## 주요 시스템

- 클래스 3종: 방랑자, 수호자, 추격자
- 구역 3종: 균열 입구, 침묵 늪지, 심연 핵
- 도전 모드 4종: 일반, 굶주린 심연, 유리 생존자, 빠른 침식
- 레벨업 능력과 진화 스킬
- 유물 8종과 빌드 조합 보너스 3종
- 심연 조각 보상과 영구 성장 3종
- 구역별 보스와 시간대별 웨이브
- localStorage 기반 저장 데이터 호환

## 조작 방법

- 모바일: Canvas를 누르고 드래그해 이동
- PC 마우스: Canvas 드래그 이동
- PC 키보드: WASD 또는 방향키 이동
- 하단 `Ⅱ` 버튼: 일시정지
- 하단 `↻` 버튼: 재시작

## 실행 방법

저장소 루트의 `index.html`을 브라우저에서 열면 실행됩니다. GitHub Pages에서는 저장소 루트를 그대로 배포 기준으로 사용합니다.

## 파일 구조

- `index.html`: 화면 구조와 일반 `<script src="">` 로딩
- `style.css`: 모바일 세로형 UI 스타일
- `data.js`: 고정 데이터, 클래스, 구역, 능력, 유물, 도전 모드
- `state.js`: 런 상태와 저장 데이터 관리
- `game.js`: 게임 업데이트와 전투 규칙
- `render.js`: Canvas 도형 렌더링
- `ui.js`: DOM, HUD, 오버레이, 입력 처리
- `main.js`: 초기화와 게임 루프

## 개발 조건

- 순수 HTML, CSS, JavaScript
- 외부 라이브러리 없음
- ES Module 없음
- npm, 번들러, 빌드 도구 없음
- 이미지 에셋 없음
- 온라인 랭킹, 계정, 네트워크 기능 없음
- PWA, 서비스워커, 사운드, 진동 없음

## 최종 점검 체크리스트

- 페이지 로드 시 콘솔 치명 오류 없음
- 시작 화면에서 클래스, 구역, 도전 모드, 영구 성장 선택 가능
- 모바일 터치 드래그와 PC 입력 가능
- 레벨업, 진화, 유물 선택 후 게임 재개
- 보스 등장, 게임오버, 클리어, 재시작 동작
- 결과 보상과 localStorage 저장 반영
- 기존 저장 데이터 누락 필드 기본값 보정
- 전체 페이지 스크롤 없음

## v1.1 보정 + v2.0 핵심 확장

- v1.1: 게임오버/클리어 결과 화면에 `다시 도전`과 `로비로` 버튼을 분리했고, 실행 중 하단 재시작 버튼 노출을 줄였습니다.
- v1.1: 초반 및 60초 이후 스폰 간격과 보스 체력/소환 주기를 소폭 상향했습니다.
- v2.0: 시작 무기 5종을 추가했습니다. 심연 탄환, 회전 칼날, 번개 사슬, 공허 지뢰, 영혼 파동을 선택할 수 있습니다.

- v2.0: 60초 이후 엘리트 적이 등장하며 보스는 체력 50% 이하에서 2페이즈로 전환됩니다.
- v2.0: localStorage 저장 구조에 `selectedWeaponId`와 `missions`가 추가되며, 기존 저장 데이터는 기본값으로 보정됩니다.

## v3.0 완성도 정리

- v2.0 전체 흐름을 유지하면서 UI 갱신 빈도, 객체 수 제한, 보상 중복 방어를 보강했습니다.
- 실행 중에는 전투 업데이트만 진행되고, 일시정지/레벨업/결과 화면에서는 입력을 즉시 정리합니다.
- 모바일 선택 카드의 아이콘 박스는 중앙 정렬을 기본으로 하고, 하단 재시작 버튼은 전투 중 숨김 처리됩니다.
- 저장 구조는 v2.0의 `version: 4`를 유지하며 새 필드는 추가하지 않았습니다.

## v3.0 테스트 체크리스트

- 페이지 로드 후 콘솔 오류가 없는지 확인
- 로비에서 클래스, 무기, 구역, 도전 모드, 영구 성장을 선택할 수 있는지 확인
- 시작 후 모바일 드래그와 PC WASD 입력이 즉시 반응하는지 확인
- 일시정지, 계속하기, 재시작, 로비로 흐름이 정상인지 확인
- 각 무기 공격, 엘리트 적, 보스 2페이즈가 정상 동작하는지 확인
- 게임오버/클리어 결과에서 다시 도전과 로비로 버튼이 정상 동작하는지 확인
- 심연 조각과 임무 보상이 중복 지급되지 않는지 확인
- 기존 localStorage 저장 데이터로 시작해도 기본값 보정 후 실행되는지 확인
- 모바일 세로 화면에서 전체 페이지 스크롤이 생기지 않는지 확인

## v3.1 무기 성장 체감 개선

- 픽토그램 박스와 글자 크기를 키우고 카드/버튼 중앙 정렬을 보정했습니다.
- 심연 탄환 외 시작 무기도 공격력, 공격 속도, 투사체 속도, 투사체 크기, 수량 계열 강화의 영향을 받습니다.
- 회전 칼날은 성장에 따라 칼날 수, 크기, 회전 속도, 피해가 증가합니다.
- 번개 사슬은 성장에 따라 번개 두께, 연쇄 수, 줄기 수, 피해가 증가합니다.
- 공허 지뢰는 폭발 범위, 설치 수, 피해가 성장하며 폭발/심연 계열 빌드 보너스를 받습니다.
- 영혼 파동은 범위, 추가 파동 수, 피해가 성장하며 일반 강화와 태그 보너스를 받습니다.
- localStorage 저장 구조는 변경하지 않았고, 무기별 성장값은 현재 런 안에서만 관리합니다.

## v3.2 맵 확장 + 카메라 시스템

- Canvas viewport는 기존 모바일 세로형 `360x560` 기준을 유지합니다.
- 실제 이동 가능한 world 크기를 `1080x1680`으로 확장했습니다.
- player, enemy, projectile, gem은 world 좌표로 관리하고 render 단계에서 camera offset을 적용합니다.
- 적은 현재 camera viewport 주변의 world 좌표에 생성됩니다.
- localStorage 저장 구조는 변경하지 않았습니다.
